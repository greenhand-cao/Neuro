## 产品功能详述与扩展

以下对原始文档中的功能点进行归类说明，并在原有描述基础上详细展开补充。

### 1. Neuro Connector 浏览器插件（信息抓取模块）

**原文要点：** “类似 zotero connector 的浏览器插件，我们的可以叫 Neuro connector，当我们浏览网页时会出…等信息的抓取与条目自动保存…用户可以输入条目命名/标签/简记等信息。”

**扩展内容：**

- **功能描述：** Neuro Connector 是一款跨浏览器（Chrome/Firefox/Edge）扩展插件，启用后可自动识别当前页面类型（网页、学术文献、视频页面等）。用户点击插件按钮或通过快捷键触发，即可抓取当前页面的标题、URL、摘要、封面图等元数据，并将其保存到 Neuro 应用的“Entry”模块中。抓取同时支持自动下载文件（如 PDF、PPT）或视频链接。
- **可输入信息：** 在保存时，弹出简易表单让用户填写或编辑条目的名称、标签（关键词）、简要笔记等。比如在学术网站上，用户可以对文献条目添加关键词标签如“文献1”“研究方法”；在旅游资讯页面，添加“城市攻略”等标签。条目类型支持区分：网页、文献、视频等，可以为不同类型定制保存字段。
- **使用场景：** 适用于收集资料、做笔记或建构知识库。例如研究者在网上查找参考资料时，通过插件快速保存文献信息；学生规划旅行时用插件收集景点和路线资讯；竞赛团队收集参考案例等。
- **技术实现：** 利用浏览器扩展 API 实现内容脚本、后台脚本与 Neuron 桌面程序（或后台服务）通信。扩展可内嵌 OCR 模块或调用接口提取网页正文。对于视频，可抓取视频页面的元数据，或借助视频平台 API 获取标题和描述。
- **数据同步：** 插件将条目信息（包含用户补充的标签）通过本地网络请求发送给 Neuro 桌面程序或云端后端，保存到对应项目的数据库。若实现跨设备同步，可考虑使用加密云存储，将用户条目同步至私人云盘或服务器。
- **错误处理：** 若抓取失败，提示用户检查网络或插件配置。保存失败时要提供重试或本地缓存方案。
- **安全隐私：** 插件本质只抓取用户浏览的公开信息，不收集用户隐私。若需上传条目到云端，应获得用户同意并加密传输，遵循最小化原则。用户标签和笔记为个人数据，建议加密存储。
- **扩展建议：** 支持批量抓取（一次性保存当前标签页所有打开的页面）、定制化抓取规则（用户可手动选取内容区域），增强的标签智能推荐（基于条目主题自动建议标签）。插件可设置同步功能，在启动Neuro时自动从浏览器接收待保存条目。

### 2. Entry 模式（项目源文件与条目管理）

**原文要点：** “当用户新建一个 Nuero 项目，此时系统自动创建项目1/Entry 文件夹…第一步中的原始信息将会保存到 Nuero 软件的项目…Entry下创建文献分类1和文献分类2文件夹，分别保存不同文献…最多三层。” “Entry 模式中允许用户上传 PPT/PDF/MP4 等文件，并自动为用户创建条目文件，允许用户移动条目，为其设置标签等。”

**扩展内容：**

- **概述：** Entry 模式相当于项目的资源库或文件管理器。每个项目创建时系统自动在内部存储（或文件系统）生成对应的目录结构，如 `项目名/Entry/`。该结构内可自动生成一级或多级子文件夹（最多三级，例如“文献”、“图片”、“视频”等分类），以便分类管理不同条目。
- **自动文件保存：** 用户通过 Neuro Connector 或手动上传的文件（PPT、PDF、MP4、Word、图片等），将被存储到当前项目的 Entry 文件夹。系统根据文件类型自动创建对应条目：例如上传一篇 PDF 文献，生成一个文献条目页面；上传视频则生成视频条目，并提取视频元数据。条目信息记录在后台数据库中，并可在前端 UI 中显示文件名、标签、创建时间等属性。
- **条目操作：** 在 Entry 模式中，用户可以以拖拽或菜单命令形式移动条目至不同分类文件夹，或新建分类文件夹来自定义组织结构。支持为条目添加多级标签和关键词，以便快速检索。可多选条目批量操作，如批量删除、移动或打标签。
- **集成工具：** Entry 页面集成了**PDF阅读器**和**翻译器**：用户点击 PDF 文件即可在内置浏览器视图中阅读，支持高亮、笔记和缩放。对于外语文档，可一键调用在线或内置翻译功能进行全文翻译（翻译结果同样可作为注释保存）。视频条目可嵌入视频播放器，提供截取视频片段和截图功能。
- **底层实现：** 可采用本地文件系统管理（例如在用户目录下创建项目文件夹）或数据库存储。若使用文件系统，需考虑权限控制和跨平台路径问题。每个条目对应一条数据库记录，记录字段包括条目类型、名称、路径、标签列表、备注、创建/修改时间等。
- **数据流：** 插件抓取或用户上传触发文件到 `Entry` 目录，Neuro 后端监听新文件事件，解析元数据并更新数据库。界面实时刷新显示新条目。
- **性能指标：** Entry 模式需能流畅管理成百上千条目，推荐使用分页或虚拟列表加载。文件预览（PDF、图片）应尽量异步加载，避免界面卡顿。搜索功能能在大型条目库中快速定位关键词，满足实时响应（<200ms）。
- **错误处理：** 提示非法文件类型或解析错误。若文件损坏或格式不支持，提示用户并记录到日志。上传大文件时提供进度显示和可取消选项。
- **安全与权限：** 文件存储于本地或云端，应设置访问权限。若多人协作，应实现访问控制、版本控制和冲突解决策略。
- **扩展建议：** 提供条目模板生成功能，例如为不同学科文献、项目里程碑、研究笔记等创建预定义表单。增加全文搜索和标签云视图，以提升内容查找效率。

### 3. AI 主节点 Markdown 生成功能

**原文要点：** “启用 AI 主导的主节点 markdown 文件生成功能，把上述的条目信息转化为主节点 markdown 格式… DOI、引用关键词、网址、访问时间、语言、文库编目、卷次、页码、出版物、ISSN、刊物、添加日期、修改日期、用户导入时创建/在 E…”

**扩展内容：**

- **功能概述：** 主节点 Markdown 文件是 Neuro 项目的核心文档，用于汇总和组织条目信息。该功能利用 AI 辅助自动生成一个结构化的 Markdown 文档，其中包含所有已保存条目的索引和摘要。具体字段包括：条目标题、作者、出版信息（DOI、ISSN、出版物名称、卷次、页码等）、检索关键词、来源网址、访问时间、语言、数据库分类号（如图书馆分类）、添加/修改日期等元数据，还可能包括用户的标签和简要备注。
- **生成方式：** 在用户完成 Entry 模式的资源整理后，系统自动或在用户命令下触发主节点生成。AI 模块（例如调用 GPT-4 或国产大模型）读取数据库中的条目信息和内容摘要，并按照预设模板填充 Markdown。用户可自定义模板格式，如章节划分、引文样式等。
- **使用场景：** 便于快速查看项目概要、编写报告或撰写论文。生成的 Markdown 可在 Neuro 内部编辑器或外部 Markdown 编辑器中继续补充内容。通过将条目汇总到一个文档，用户能够看到项目的全貌和细节，例如开题报告或项目计划文档所需的参考资料列表。
- **技术实现：** 可先通过访问 CrossRef、CNKI 等文献检索接口获取条目 DOI 和出版信息，也可调用 AI 读懂并补全字段。AI 通过自然语言处理技术提取每个条目的关键信息并填充表格结构。
- **示例格式：**
  ```markdown
  ## 项目: 旅游规划2025

  ### 文献1: 《智能旅行助手研究》  
  - **作者：** 张三  
  - **出版物：** XX学术杂志, 2022年, 卷 5, 页码 10-20  
  - **DOI：** 10.xxxx/abcd1234  
  - **关键词：** 旅游规划, AI助手  
  - **来源网址：** http://...  
  - **添加日期：** 2025-06-01
  - **用户标签：** 旅游, 规划  
  - **摘要：** AI系统在旅游行业的应用...
  ```
- **性能指标：** 生成主节点 Markdown 所需时间应在可接受范围内（如<10秒，具体视条目数量而定）。AI 处理可以异步进行，并在结果准备好后通知用户。用户可选择部分条目生成或全部汇总。
- **错误和验证：** 若 AI 未能识别某些字段，应标记为“未知”并允许用户手动编辑补充。生成后，用户可在预览界面对结果进行审校并保存。
- **接口与可扩展性：** 允许用户导入外部引用管理器（如 Zotero、EndNote）的 BibTeX/CSV 文件，通过插件形式扩展对其他格式的支持；也可导出 Markdown 结果。
- **扩展建议：** 加入引用格式化支持（如自动生成参考文献列表）、跨条目自动链接（将相似主题或关键字条目互相引用）。增加版本控制功能以追踪主节点文件的更新历史。

### 4. Thinking 模式（AI分析与标注模式）

**原文要点：** “Entry 模式编辑完成之后，进入项目最具特色的 thinking 模式（或叫 AI 打标）。思维模式…如果 AI 需要进一步查询可以打开跳转链接进一步理解。为了节省 token，如无必要，AI 不会主动打开跳转链接对应文件。”

**扩展内容：**

- **模式介绍：** Thinking 模式是 Neuro 的智能分析核心，类似于 AI 助理在项目资源的基础上生成深入分析和标注。在此模式下，系统调用 AI（本地模型或云端大模型）对 Entry 中的条目内容进行处理，生成可视化标注、摘要和智能建议。
- **操作流程：** 用户从 Entry 切换到 Thinking 模式界面，系统列出待分析的条目列表。用户可以选择全部或部分条目进行分析。点击“开始分析”后，AI 模型逐条读取文本、笔记或文档内容，执行以下任务：
  - **内容提炼：** 对文献、PPT、视频转录文本进行摘要提炼，自动生成每个条目的**关键信息摘要**。
  - **标签标注：** AI 自动根据内容给条目添加主题标签，构建标签层级（例如将“人工智能”归类到“技术”类别）。
  - **问题回答：** 对每个条目内的信息生成问答，例如提取“电极制备材料/电极阻抗/修饰材料”参数，存入数据字段中，便于后续检索。
  - **关联链接：** 尝试解析每个条目的引用链接和参考文献，对可访问的资源做自动跳转（如到外部数据库抓取更多信息）以补充内容。如果需要额外数据，AI可按需使用集成的搜索接口（注意遵守token成本考虑）。
- **视频关键帧处理：** 对于视频条目，AI 将自动**抽取关键帧**。例如，识别视频中出现的幻灯片界面或关键词出现的时刻，截取帧作为静态图片，并保存在与该视频条目同一文件夹下。每个关键帧可附上简要说明或自动提取的文字片段，丰富视频条目的元数据。这样用户无需手动翻看即可快速预览视频重点。
- **可视化输出：** 分析完成后，系统生成可视化报告，包括思维导图、标签云、关联网络图等，显示条目之间的主题关联和信息结构。用户可在 Thinking 模式界面查看、编辑AI生成的节点和连线。
- **AI对话接口：** 在 Thinking 模式中内置 AI 聊天助手，用户可以针对当前分析项目自由提问。系统会优先搜索当前项目的相关 Markdown 文档和条目数据作为上下文（例如在文献条目里搜索关键词“电极阻抗”），然后结合AI推理给出回答。这样可快速得到项目相关数据与见解。
- **技术实现：** 可使用 OpenAI GPT 系列或国产大模型（如讯飞Spark、华为盘古等），结合专门的检索增强技术（RAG）来处理项目文档。视频处理可集成 FFmpeg 截帧与 OCR 识别。前后端通过WebSocket或REST API完成数据交互。
- **性能要求：** 分析过程通常较耗时，应在前端提供进度条和日志输出。对于大型项目，可设计分批处理机制。
- **错误容忍与用户引导：** 若AI分析结果不满意，用户可以手动调整或重新运行部分分析。建议Thinking模式支持用户插入新的提示词或反馈来校正AI输出（例如为GPT提供附加说明以提高准确度）。
- **扩展建议：** 增加“深度对话”功能，让用户与AI就项目内容展开多轮探讨，AI根据用户反馈逐步完善项目思路；或者开放插件接口，让第三方可接入不同领域的智能分析模块。

## 性能与可靠性

- **性能指标：** 在 200 条目以下的典型项目中，条目搜索应<0.2秒响应；条目创建/移动等基本操作实时生效。对于大型项目，搜索和重建视图可异步进行并预留进度条。PDF/PPT 解析、视频关键帧提取等操作并非实时，需异步后台处理并通知用户完成。
- **稳定性与容错：** 防止长任务（如视频处理、批量AI分析）阻塞界面。关键功能需自动保存和断点续传；数据库损坏时提供修复工具。
- **异常处理：** 所有模块应捕获异常并记录日志。比如分析失败时要提示“分析超时或发生错误”，并在日志中保存错误详情供开发调试。

## 错误处理与用户反馈

- **数据校验：** 上传文件时校验格式合法性（如视频格式支持列表、扫描病毒），并在失败时提示用户。
- **操作确认：** 对删除条目、清空项目等危险操作弹出确认对话框，避免误操作。
- **日志与报告：** 系统自动记录运行日志。对于 AI 调用或同步错误，可生成诊断报告提示用户或技术人员。
- **用户反馈渠道：** 界面内置反馈入口，用户可提交错误描述和截图。

## 安全与隐私

- **个人信息保护：** Neuro 设计时必须遵守《个人信息保护法》等法规要求：只收集实现功能所需的最小信息，明确处理目的并告知用户。例：如果需要联网调用 AI API，应告知用户并获得同意，确保用户可撤回同意。
- **数据加密：** 重要数据（用户笔记、标签、聊天记录等）建议在本地数据库中加密存储。对于云端同步的条目数据，使用 HTTPS+TLS 传输并加密服务器存储。

## 示例操作流程

以**旅游规划项目**为例，说明用户如何使用 Neuro 完成一个流程：

1. **创建项目：** 打开应用，新建项目“欧洲自由行2025”，系统自动生成 `Entry/` 文件夹。
2. **收集资料：** 使用 Neuro Connector 浏览旅游攻略网站、景点介绍视频，将它们保存为条目并分类到“欧洲景点”或“酒店/机票”子文件夹。上传已下载的 PDF 旅游资料。
3. **管理条目：** 在 Entry 模式查看保存内容，为各条目补充标签（如“巴黎”“预算”），上传相关照片或地图截图。
4. **生成节点文档：** 点击“生成主节点 Markdown”，AI 汇总每个条目的信息在其目录下生成markdown文件，这个叫做每个条目具有的总结该条目所有信息的即主节点markdown文件，作为后续AI查询与修改的首选文件。
5. **AI 分析：** 切换到 Thinking 模式，让 AI 阅读所需的主节点markdown文件，提炼出行程要点、生成时间线（例如每天行程），并回答用户问题如“预计旅行时长和总费用”。
6. **可视化呈现：** 进入可视化选项卡，AI会根据主节点markdown文件中的标签内容准备所需数据并生成可视化图形，查看旅行路线地图和甘特图，确认行程合理。
7. **任务跟踪：** 在Entry模式使用“任务制定”功能，为每个任务创建模板型的主节点markdown文件，引导输入准备任务（订机票、签证、预定酒店等）。输入完成之后打开Follow模式，用户只需要和AI对话，即可自动标记已完成的任务。同时AI 根据进度更新日程提醒用户下一个未完成项，如“还需要预定博物馆门票”。
8. **完成汇报：** 根据 Neuro 提供的报告和可视化图表，用户生成最终行程计划，项目圆满完成。
